Build a new feature in my existing Base44 app called:

ERCOT Congestion Intelligence Engine

Purpose:
Create a portfolio-grade power market analytics feature that uses the ERCOT zonal, LMP-level, settlement point, node-level, and bus-level data already available in the app to analyze, predict, and explain congestion and basis risk. The feature should also include a simplified PyPSA-based simulation layer if technically feasible.

Primary objective:
Use existing app data from 2024 onward as the source of truth. Do not use data before 2024. If any required ERCOT data is missing from the app, retrieve or reference the missing data from ERCOT public market data sources, but only for 2024 onward.

Important data rule:
Use app data first. Only supplement missing fields from ERCOT public data. Keep a data-source flag on every record showing whether the data came from:

1. Existing app data
2. ERCOT supplemental data
3. Derived/calculated field

Feature scope:

1. Historical Congestion Analytics

Use the available ERCOT data to calculate and display:

* Hub price
* Load zone price
* Node/bus LMP
* Settlement point price
* Node-to-hub basis
* Zone-to-hub basis
* Node-to-zone basis
* DAM vs RTM spread
* Congestion component, if available
* Loss component, if available
* Price adder, if available
* Negative price events
* Price spike events
* Volatility by node, zone, hub, and settlement point
* Frequency of congestion events by hour, day, month, and season

Create a calculated congestion event flag using a configurable threshold:

* Congestion event = absolute node-to-hub basis greater than user-selected threshold, default $10/MWh
* Severe congestion event = absolute node-to-hub basis greater than $25/MWh
* Extreme congestion event = absolute node-to-hub basis greater than $50/MWh

Allow users to change these thresholds in the UI.

2. Congestion Prediction Model

Build a forecasting model that predicts congestion and basis risk.

Model targets:

* Probability of congestion event
* Expected node-to-hub basis
* Expected zone-to-hub basis
* Expected DAM vs RTM spread
* Probability of negative pricing
* Probability of price spike
* Volatility risk score

Training and testing:

* Train only on 2024 and 2025 data.
* Treat 2026 as an out-of-sample backtest period.
* Do not use 2026 actuals for model training.
* Generate predictions for 2026 timestamps where actuals exist.
* Compare 2026 predicted values to 2026 actuals already in the app.
* If 2026 actuals are missing, supplement from ERCOT public data, but only for 2026 dates that are available.

Suggested model features:

* Hour of day
* Day of week
* Month
* Weekend/weekday flag
* Market type: DAM or RTM
* Hub
* Load zone
* Node/bus
* Settlement point
* Prior interval LMP
* Prior interval basis
* Rolling average basis
* Rolling volatility
* Historical same-hour average basis
* Historical same-hour percentile
* Load, if available
* Wind generation, if available
* Solar generation, if available
* Temperature, if available
* Outage/derate information, if available
* Scarcity or price adder, if available

Use a practical modelling approach:

* Start with baseline models such as historical percentile, moving average, and seasonal/hourly averages.
* Then add machine learning models such as Random Forest, Gradient Boosting, XGBoost, or LightGBM if available.
* Use explainability where possible, such as feature importance or SHAP-style explanations if supported.
